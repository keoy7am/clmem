## WS-1: Memory Intelligence

**Files**: `src/tui/process_list.rs`, `src/tui/mod.rs`

### 1-A: RSS Delta 欄位

在 `ProcessListPanel` 追蹤每個 PID 的前一次 RSS，顯示 delta：

- 新增 `previous_rss: HashMap<u32, u64>` 欄位
- 每次 `update()` 時計算 delta = current_rss - previous_rss
- 新增 "Delta" table 欄位，顯示格式：
  - `+2.5 MB` (紅色，正成長)
  - `-1.2 MB` (綠色，回收)
  - `--` (首次觀測或無變化)
- Table 寬度調整：Delta 欄位 `Length(10)`

### 1-B: 記憶體顏色分級

RSS 欄位依絕對值上色（取代目前白色）：

```
< 50 MB  → Color::Green
50-200 MB → Color::Yellow
> 200 MB  → Color::Red
```

VMS 欄位同理但閾值更高（< 1GB / 1-5GB / > 5GB）。

### 驗證

- `cargo clippy -- -D warnings && cargo test`
- TUI 啟動後觀察 Delta 欄位隨時間變化
- 確認 delta 在 daemon 重啟後歸零（首次觀測）

## WS-4: Inline Sparkline（RSS 趨勢）

**Files**: `src/tui/process_list.rs`, `src/daemon/mod.rs`, `src/models/snapshot.rs`

### 資料面

目前 `MemorySnapshot.processes` 只有當下值，沒有歷史。方案：

**方案 A（TUI 端累積）**：
- `ProcessListPanel` 內維護 `rss_history: HashMap<u32, VecDeque<u64>>`
- 每次 `update()` 時 push 當前 RSS，保留最近 20 個樣本
- 不需修改 daemon 或 IPC protocol

**選擇方案 A**（最小侵入性，不動 daemon）

### 渲染

在 table 新增 "Trend" 欄位（`Length(12)`），使用 Braille block 字元手動渲染：

```rust
fn render_sparkline(history: &[u64]) -> String {
    let bars = ['▁', '▂', '▃', '▄', '▅', '▆', '▇', '█'];
    let max = history.iter().max().copied().unwrap_or(1).max(1);
    history.iter()
        .map(|&v| {
            let idx = ((v as f64 / max as f64) * 7.0) as usize;
            bars[idx.min(7)]
        })
        .collect()
}
```

欄位會顯示如：`▁▂▃▅▇██▇▅▃` 表示先升後降。

### 驗證

- Sparkline 在 TUI 啟動 20 秒後開始出現完整趨勢
- 新 process 出現時 sparkline 逐漸累積
- Process 消失後 history 自動清除

## 欄位 Layout 總覽 (v0.8.0)

```
| PID(8) | Command/Name(Min40) | RSS(10) | Delta(10) | VMS(10) | Trend(12) | State(8) | Uptime(10) |
```

固定寬度欄位: 8 + 10 + 10 + 10 + 12 + 8 + 10 = 68
Command 彈性最小寬度: 40
總最小寬度: 108 字元（標準 terminal 120+ 足夠）
